객체 지향 프로그래밍(OOP)의 장점

  1. 1. 상속을 통해 코드 재사용성을 높일 수 있다.
  2. 2. 잘 설계된 클래스를 만들어서 독립적인 객체를 사용함으로써 생산성을 향상시킬 수 있다.
  3. 3. 일상생활의 모습의 구조가 객체에 자연스럽게 녹아 있어서 생각하는 것을 그대로 표현 할 수 있다.
  4. 4. 프로그램 수정시 추가, 수정을 하더라도 캡슐화를 통해 주변에 영향을 적게 줄 수 있으며, 이때문에 유지보수가 매우 쉽고 간편해져 경제적이다.

객체 지향 프로그래밍(OOP)의 단점

  1. 1. 객체가 처리하는 것에 대한 명확한 이해가 필요하기때문에 설계단계부터 많은 시간이 소모 된다.
  2. 2. 객체 지향 언어는 대체적으로 실행속도가 느리다.
  3. 3. 다중 상속이 지원되는 C++의 경우 너무 복잡해져 코딩의 난이도가 상승할 수 있다.

객체 지향 프로그래밍(OOP)의 특징

  1. - 캡슐화 : 입력값이 문제가 있어도 사용자가 알지 못하도록 처리하는 방법이 캡슐화 이다.
  2. - 상속 : 상속은 말그대로 상위 클래스의 모든 것을 하위 클래스에서 상속받아 사용 할 수 있다.
  3. - 다향성 : 상속과 연관이 있는 개념으로, 다양한 하위 클래스를 정의 할 수 있다.
  4. - 추상화 : 공통 속성과 기능을 추출하고, 필요없는 특성을 제거하는 과정을 말한다.

클래스(class) 란?

es6에서 추가된 OOP 클래스(class)문법이다.
이전에는 자바스크립트로 객체 지향 프로그래밍(OOP)을 하려면 prototype 기반으로 생성자 함수를 통해서 할 수 있었다.
es6는 클래스 함수를 통해 prototype 방식보다 더 명료하게 객체 지향 프로그래밍을 할 수 있게 되었다.
클래스 역시 일반 함수와 동일하게 선언식과 표현식으로 만들 수 있다.
하지만 클래스 함수는 일반 함수와 다르게 선언식, 표현식 둘다 호이스팅 되지 않는다.

메소드 정의

클래스의 몸통은 {}(중괄호)이며, 메소드는 class 내부에 정의해야 한다.
클래스 몸통은 strict mode(엄격 모드)로 실행된다.
하나의 클래스당 생성자 메소드(constructor methods)는 하나만 사용 할 수 있다.

class 클래스이름1 {
    // strict mode
    // 메소드들은 모든 하위 클래스에 상속 할 수 있다.
    // 한번 정의한 메소드는 많은 클래스에서 재사용 할 수 있다.
    constructor(parm1, parm2) {
        this.parm1 = parm1
        this.parm2 = parm2
    }
    메소드1 = function() {
        console.log("나는 메소드1 이다.");
    }
    메소드2 = function(){
        console.log("나는 메소드2 이다.");
    }
}

오토박싱(Autoboxing)

this 값이 null 혹은 undefined 일 경우 window 객체로 자동으로 변환 해주는 것을 오토박싱(autoboxing)이라고 한다.
프로토타입 기반의 생성자 함수 에서는 오토박싱(Autoboxing)이 일어나지만,
es6 클래스 기반은 오토박싱(Autoboxing)이 되지 않기 때문에 undefined가 출력 되는 것을 알 수 있다.
메소드를 변수에 저장하여 사용할 경우 this에 유의해서 사용해야 한다.

// 부모의 클래스에 있는 정의 된 내용을
// 자식 클래스에서 가져다 쓸 때 extends를 사용한다.
// 생성자 함수부터 메서드까지 모두 상속 받을 수 있다.
class 부모클래스 {
    constructor(매개변수) {
        // 이름을 서로 다르게 하지 않는 이유는
        // 굳이 다르게 쓸 필요가 없기 때문이다.
        this.매개변수 = 매개변수;
    }
 
    speak() {
        console.log(this.매개변수 + '안녕하세요.');
    }
}
 
class 자식클래스 extends 부모클래스 {
    speak() {
        console.log(this.매개변수 + '반갑습니다.');
    }
}

class 자식클래스 extends 부모클래스 {

        super.부모메서드1();
        console.log(this.매개변수 + '반갑습니다.');
    }
}
